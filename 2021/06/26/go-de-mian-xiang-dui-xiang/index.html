go的面向对象

Go
Go
发布日期:   2021-06-26
  • Go语言仅支持封装，不支持继承和多态

初始化结构体

通过type <name> struct {code}的形式定义结构体

type MyStruct struct{
  value int
}

// new struct
myStruct := MyStruct{value: 3}

var myStruct MyStrcut

node3 := new(MyStruct)

结构体的方法

要为结构体添加函数。需要将结构体作为接收参数（接收者）写在方法前面。

func (myStruct MyStruct) getValue() int {
  return myStruct.value
}

func (myStruct MyStruct) print() {
  fmt.Prtinln(myStruct)
}

// 这样getValue和print就作为了MyStruct的函数。可以直接调用
myStruct := MyStruct{value: 3}
myStruct.print

// 如果这样写
func print(myStruct MyStruct){
  fmt.Prtinln(myStruct)
}
// 则需要当作一个普通函数调用
print(myStruct)

关于值传递

在结构体方法中，接收者也是值传递。意味着myStruct.getValue()调用的其实是myStruct的复制的value。这种时候需要将接收者作为指针，传递地址。

func (myStruct *MyStruct) setValue(newValue: int) {
  myStruct.value = newValue
}

// usage

myStruct.setValue(100) // value = 100

这么做，调用setValue方法时，实际上调用的是myStruct对应地址上的方法。不过在使用没有任何区别，都是用.func调用即可。

同样的，当我们将myStruct的指针付给一个新变量时，调用这类办法，也会将myStruct的value值进行修改。

myStruct2 := &myStruct // value = 3
myStruct2.setValue(100)

fmt.Println(myStruct2) // value = 100
fmt.Println(myStruct)  // value = 100

同时，nil也可以调用指针的方法。并且方法调用本身不会报错，只当进行具体操作时，比如过nil并没有value这个属性，所以nil.value时会报错。

指针接收者还是值接收者

  • 要改变内容必须使用指针接收者。
  • 结构过大也考虑使用指针接收者。
  • 一致性：如果有指针接收者，最好其他方法也修改为指针接收者。

构造方法

Go没有构造方法一说。不过可以通过方法的形式，创建构造方法

func createMyStruct(newValue int) *MyStruct{
  return &MyStruct{value: newValue}
}

有几个点需要注意：

  • 这里返回的是一个局部变量。在方法里创造的MyStruct
  • Go的编译器会在编译时判断这个变量建立在堆上还是栈上。
    • 如果返回值是一个指针。Go会将其创建在堆上。供外部使用，同时参与到后续的垃圾回收。
    • 如果返回值是一个值。Go会将其创建的栈上。返回一个其的复制。并在结束时销毁。

包

  • 每个目录只允许有一个包
  • main包包含可执行入口。
    • 如果一个目录下有一个main函数，那么这个目录下只能有一个main包(即package main)。
  • 为结构定义的方法必须放在同一个包内。
    • 但可以在同一个包下的不同文件。

封装

在Go中，首字母大写有含义

  • 首字母大写。表示public
  • 首字母小写。表示private
  • 结构的元素，方法，结构体本身都遵循上述可见规则。

使用

使用包.类进行调用。

// 假设Node在tree包下,下述代码在use包下的hello.go
// -root
// |-use
// |--hello.go
// |-tree
// |--node.go

// 一个目录下只允许有一个package main
package main

// 需要根据目录进行import
import "awesomeProject/tree"

func main() {
  // 包名tree不能省略
    node := tree.Node{Value: 2}
    node.Print()
}
