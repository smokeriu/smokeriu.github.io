Linux中awk与sed指令的使用笔记

Linux
Linux
发布日期:   2018-02-17

最近工作要使用到这对命令进行脚本编写，使用感觉特别容易忘，故记录整理下。

awk

简单来说，awk是把文本逐行读入，默认以空格为列分隔符进行切片。对切好对数据再进行处理。
本文会再学习实验的同时，简单介绍一些常用的awk命令。

使用方法：

awk '{pattern + action}' {filename}

pattern表示awk在数据中查找对内容。
action表示找到后执行什么命令。
{}用来对一系列指令进行分组。

调用途径：

#1)命令行
awk [-F field-separator] 'commands' input-files

#2)shell脚本
#awk命令解释器作为脚本首行，相当于将#!/bin/bash => #!/bin/awk

简单使用：

首先，准备一个文本格式对文件（a1）。以空格分开。
再准备一个文件b1，以【,】作为分隔符
比如说a1对内容：第三行会多处一列。

1）区分行与列，打印数据

1）打印a1所有数据：

awk '{print}' a1
#输出：
first line 3 4
second second 3 4
hadoop 3 4 50

2）打印a1第一列，第二列数据：

awk '{print $1,$2}' a1
#输出：
first line
second second
hadoop 3

#注意,如果不加[,]
awk '{print $1$2}' a1
#则输出
firstline
secondsecond
hadoop3

#注意,如果不加[$]
awk '{print $1 2}' a1
#则输出
first2
second2
hadoop2

3）打印第1列第1+行数据：使用内置的NR变量。（文章结尾会总结下常用的内置变量）

awk 'NR>1{print $1}' a1
#结果：
second
hadoop

4）如果需要打印第2+列，就不那么友好了，不过也可以使用循环来完成：
打印第2+列的数据

awk '{for(i=1;i<=2;i++)$i="";print}' a1
#结果：
  3 4
  3 4
  4 50

2）分隔符/正则查找

awk默认是识别空格为列分隔符，不过比如csv使用逗号来进行风格的，显然需要指定分隔符。这里会使用：-F fs。-F就是指定输入文件分隔符，fs可以是一个字符串，也可以是一个正则表达式

b1内容如下：

1,2,3,4,5,6
a,b,c,d,e,f,g
a,s,d,f,g,h,j,k
a,a,a,a,a,a,a,a
b,b,b,b,b,b,b,b

1）读取以逗号为分隔符的b1文件，打印第2，3行，第1，2列：

awk -F ',' 'NR>1 && NR<4{print $1,$2}' b1
#结果：
a b
a s

2）新建c1：文本内容：

a,1,ok
b,2,no
c,3,ioki

找到ok，并打印第1列：

awk -F ',' '/ok/{print $1}' c1
#结果：
a
c

awk常用内置变量

变量 含义
ARGC 命令行参数个数
ARGV 命令行参数排列
ENVIRON 支持队列中系统环境变量的使用
FILENAME awk浏览的文件名
FNR 浏览文件的记录数
FS 设置输入域分隔符，等价于命令行 -F选项
NF 浏览记录的域的个数
NR 已读的记录数
OFS 输出域分隔符
ORS 输出记录分隔符
RS 控制记录分隔符
1）使用：读取文件a1，统计文件名，行号，列数
```shell
awk ‘{print “name:”FILENAME”,line:”NR”,cols:”NF}’ a1
#结果
name:a1,line:1,cols:4
name:a1,line:2,cols:4
name:a1,line:3,cols:4
```
#### 结合其他命令行
awk可以结合其他命令行返回第结果进行处理。通过 风格。awk第用法和上面将第是一样的。
1）利用awk统计最近5个登陆的用户的用户名和ip或主机名。（last后面不加指令)
```shell
last awk ‘NR<5{print $1,$3}’
#结果
hadoop 192.168.199.203
hadoop 192.168.199.203
hadoop 192.168.199.203
hadoop 192.168.199.203
hadoop 192.168.199.203
```

sed

sed是一种在线编辑器，每次处理一行内容。它将处理的行放入临时缓冲区，然后使用sed命令处理缓冲内容。默认会将处理好的内容发往屏幕，可以通过重定向，将内容输出至文件。
sed主要用来自动编辑1～多个文件。
特别的，sed是一行一行处理的。

语法：

sed [nefr] [action] [file]

部分：
| nefr | 含义 |
| —- | ———————- |
| -f | 使用file里到sed命令 |
| -i | 直接将结果修改到file里 |

action：[n1[,n2]] func
| func | 含义 |
| —- | ———————- |
| a | 新增行（在该行后插入） |
| c | 修改行 |
| d | 删除 |
| i | 插入行（在该行前插入） |
| s | 取代（替换） |

简单使用：替换

1）将hello,hello,world的第一个hello替换为gwkki

#语法：'s/pattern/replacement/flag'
echo "hello,hello,world" | sed 's/hello/gwkki/'
#结果
gwkki,hello,world
#特别的，这个替换会作用到每行第一个匹配到的。

2）所有hello替换为gwkki

echo "hello,hello,world" | sed 's/hello/gwkki/g'
#结果
gwkki,gwkki,world

3）替换第二个hello

echo "hello,hello,world" | sed 's/hello/gwkki/2'
#结果
hello,gwkki,world

说下flag：
数字表示替换第几处匹配到的。
g表示全部替换
p表示匹配的行会被额外打印。
w表示保存到指定文件（读取文件可以用 sed -i command来修改并保存）
4)修改c1中的xxx为yyy并保存：

sed -i s/xxx/yyy/ a1

操作某些行：

#操作第二行：
sed '2s/xxx/yyy/' a1

#操作2-4行：包含2，4
sed '2,4s/xxx/yyy/' a1

#操作3+行：包含3
sed ‘3，$s/xxx/yyy/’ a1

删除

#删除匹配到k到行：
sed '/k/d' c1

#删除第3-5行：
sed '3,5d' c1

插入新增

#在"line 1"的下一行新增line2
echo "line 1" | sed 'a line 2'
#在"line 1"的上一行新增line2
echo "line 1" | sed 'i line 2'

替换特殊字符 如：’

使用””扩起来即可：“可以用"匹配到

#将'替换成{}
sed "s/\'/{}/" a1
