Typesafe的ConfigFactory的简单使用

Utils
Utils
发布日期:   2017-11-26

ConfigFactory

ConfigFactory能够帮助我们快速的读取配置文件，帮助我们修改一些配置信息，而不用修改代码。
例如我们可以将sql的连接信息放入配置文件中，而不是写死在代码里。
类似于Java的properties，不过ConfigFactory支持多种文件格式，包括：

The convenience method ConfigFactory.load() loads the following (first-listed are higher priority):
a) system properties
b) system properties application.conf (all resources on classpath with this name)
c) application.json (all resources on classpath with this name)
d) application.properties (all resources on classpath with this name)
e) reference.conf (all resources on classpath with this name)

其他的优点，用途，用法大家可以参见github项目。
里面的README.md进行了详细的说明。

使用

引入依赖：

需要注意：1.2.1及以前的版本是基于Java6开发的，而1.3及之后的版本都需要基于1.8。

Version 1.2.1 and earlier were built for Java 6, while newer versions(1.3.0 and above) will be built for Java 8.

以maven为例：

<dependency>
    <groupId>com.typesafe</groupId>
    <artifactId>config</artifactId>
    <version>1.3.2</version>
</dependency>

获取配置文件


使用load方法可以加载到实现准备好的配置文件，默认加载classpath下的application.conf，application.json和application.properties文件。当然也可以调用ConfigFactory.load(confName)加载指定的配置文件。
获取到之后通过getxxx方法，就能够提取里面的值了。本文主要介绍下简单使用，更复杂的使用请参考:github项目。

常见配置文件的书写规则：

上面提到过，ConfigFactory支持多种配置文件格式，常见的有application.conf，application.json和application.properties文件。

application.conf

一般使用k=v的形式即可：

db.default.url="jdbc:mysql://localhost:3306"
db.default.name="dbname"
db.default.passwd=123456
//使用
conf=ConfigFactory.load(confName)
conf.getString("db.default.url"))
conf.getInt("db.default.passwd"))

也可以使用这种配置形式：

mysql-app {
    url="jdbc:mysql://localhost:3306"
}
//使用
conf=ConfigFactory.load(confName)
conf.getString("mysql-app.url"))

application.json

{
    "db":{
    "url":"jdbc:mysql://localhost:3306"
    "dbname":"dbname"
    }
}
//使用
conf.getString("db.url"))
conf.getString("db.dbname"))

application.properties

db.url=jdbc:mysql://localhost:3306
//使用同上。

简单使用：

object mysqlApp {
  def main(args: Array[String]): Unit = {
    val spark = SparkSession.builder()//.appName(this.getClass.getSimpleName).master("local[*]")
      .getOrCreate()

    if(args.size != 2){
      println("args error////inputPath///confPath")
      System.exit(1)
    }

    val Array(intput,conf) = args

/**
 * 文件内容：
 * a,1,A
 * b,1,B
 * c,1,C
 */    
    val readDF = spark.read.format("text").load(intput)
    import spark.implicits._
    val df = readDF.rdd.map(x => {
      val splits = x.getString(0).split(",")
      (splits(0), splits(1), splits(2))
    }).toDF("col1", "col2", "col3")

    df.printSchema()
    df.write.options(getDBConf(conf))
      .mode("overwrite")
      .format("jdbc").save()

    def getDBConf(confPlace:String="application.conf")={
      val config = ConfigFactory.load(confPlace)
      val url = config.getString("db.default.url")
      val configMap = HashMap[String, String](
        ("url", config.getString("db.default.url")),
        ("db", config.getString("db.default.name")),
        ("dbtable", config.getString("db.default.table")),
        ("user", config.getString("db.default.user")),
        ("password", config.getString("db.default.password")),
        ("driver",config.getString("db.default.driver"))
      )
      configMap
    }
  }
}
